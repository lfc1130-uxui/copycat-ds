Button

Triggers actions or navigation. Comes in 4 variants, 3 sizes, and 5 interactive states — all wired to design tokens.

Interactive Demo
All Variants
Props
NameTypeDefaultDescription
variant 'primary' | 'secondary' | 'ghost' | 'danger' 'primary' Visual style of the button
size 'sm' | 'md' | 'lg' 'md' Height and padding scale (32 / 40 / 48px)
disabled boolean false Prevents interaction. Applies neutral gray colors — does not use opacity dimming.
icon IconName — Name of the icon to render inside the button (e.g. "search", "trash")
iconPosition 'leading' | 'trailing' 'leading' Position of the icon relative to the label
loading boolean false Replaces content with a spinner and disables interaction. Button keeps its natural width.
fullWidth boolean false Stretches button to fill its container
type 'button' | 'submit' | 'reset' 'button' Native HTML button type
children ReactNode — Label, icon, or any React content
Usage
TSX 
import { Button } from '@copycat/design-system';

// Variants
<Button variant="primary">Save changes</Button>
<Button variant="secondary">Cancel</Button>
<Button variant="ghost">Learn more</Button>
<Button variant="danger">Delete account</Button>

// Sizes
<Button size="sm">Compact</Button>
<Button size="lg">Prominent</Button>

// With icon
<Button variant="primary" icon="search">Search</Button>
<Button variant="primary" icon="download" iconPosition="trailing">Download</Button>

// States
<Button loading>Saving…</Button>
<Button disabled>Unavailable</Button>
<Button fullWidth>Full width</Button>

Input

Single-line text field with 7 validation states, icon slots, prefix/suffix, password toggle, and full token coverage. Built for banking-grade form flows.

Interactive Demo
All States
Props
NameTypeDefaultDescription
size 'sm' | 'md' | 'lg' 'md' Height scale — 32 / 40 / 48px
validationState 'default' | 'error' | 'success' | 'loading' 'default' Controls border color and the trailing icon slot
label string — Label rendered above the field
helperText string — Hint text below the field. Always visible unless replaced by errorMessage
errorMessage string — Red error text shown below when validationState="error"
leadingIcon IconName — Icon rendered inside the left edge of the field
trailingIcon IconName — Icon rendered inside the right edge. Overridden by validation state icons
prefix string — Inline text before the field (e.g. "$"). Sits flush, no divider
suffix string — Inline text after the field (e.g. "USD"). Separated by a left border
fullWidth boolean false Stretches the input to fill its container
disabled boolean false Uses color/bg/disabled and color/text/disabled tokens — no opacity dimming
readOnly boolean false Subtle background, default border — signals non-editable but readable
required boolean false Appends a red asterisk to the label
type string 'text' Native input type. password adds a show/hide toggle in the trailing slot
Usage
TSX 
import { Input } from '@copycat/design-system';

// Basic
<Input label="Account number" placeholder="Enter 10-digit number" />

// Validation states
<Input validationState="error"   errorMessage="Account not found" />
<Input validationState="success" helperText="Account verified" />
<Input validationState="loading" helperText="Checking…" />

// With icons and prefix/suffix
<Input leadingIcon="search" placeholder="Search transactions" />
<Input prefix="$" suffix="USD" placeholder="0.00" />

// Password with show/hide toggle
<Input type="password" label="Password" helperText="At least 8 characters" />

// Disabled / Read-only
<Input disabled />
<Input readOnly defaultValue="ACC-00129834" helperText="Contact support to change" />

Select

Custom listbox dropdown with 6 validation states, optional leading icon, 3 sizes, and full keyboard navigation. Click any trigger below to open the live dropdown.

Interactive Demo
All States
Props
NameTypeDefaultDescription
options SelectOption[] — Array of { value, label, disabled? } objects
value string — Controlled selected value
defaultValue string — Initial value for uncontrolled usage
onChange (value: string) => void — Called when the user selects an option
size 'sm' | 'md' | 'lg' 'md' Height scale — 32 / 40 / 48px
validationState 'default' | 'error' | 'success' | 'loading' 'default' Controls border color and the trailing icon slot
label string — Label rendered above the field
placeholder string 'Select…' Tertiary-colored text shown when no value is selected
helperText string — Hint text below the field
errorMessage string — Red error text shown when validationState="error"
leadingIcon IconName — Icon rendered inside the left edge of the trigger
fullWidth boolean false Stretches the trigger and dropdown to fill its container
disabled boolean false Uses color/bg/disabled and color/text/disabled tokens — no opacity
required boolean false Appends a red asterisk to the label
name string — Hidden <input> name for form submission
Usage
TSX 
import { Select } from '@copycat/design-system';

const options = [
  { value: 'current',  label: 'Current account — ••4582' },
  { value: 'savings',  label: 'Savings account — ••1293' },
  { value: 'isa',      label: 'Cash ISA — ••8801' },
];

// Basic uncontrolled
<Select label="Account" placeholder="Select account" options={options} />

// Controlled
<Select label="Account" value={value} onChange={setValue} options={options} />

// Validation states
<Select validationState="error"   errorMessage="Please select an account" options={options} />
<Select validationState="success" helperText="Account verified" options={options} />
<Select validationState="loading" options={options} />

// With leading icon
<Select leadingIcon="globe" label="Country" options={options} />

// Disabled / Required
<Select disabled options={options} />
<Select required label="Account" options={options} />
